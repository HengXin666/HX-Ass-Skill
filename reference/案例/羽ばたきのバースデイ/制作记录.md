## 制作流程总结

### 1. 输入文件

- **K帧歌词**: `reference\案例\羽ばたきのバースデイ\k帧.ass`
  - 由 LDDC 生成, 包含 `orig` (日语) 和 `ts` (中文翻译) 两轨
  - K帧标签为 `\kf` 格式(需转换为 `\k`)
  - 共 210 行(含元数据)

- **参考模板**: 天使の3P 原版 KFX(9858行, 含完整 code/template/fx)
  - 6 组 Style(OPJP, OPJP 2, OPJP 3-1, OPJP 3-2, OPJP 4, OPJP 5)+ OPCN
  - 每组 Style 有对应的 `-furigana` 变体
  - 包含贝塞尔曲线、蝴蝶翅膀、音符粒子等高级效果

- **注音映射**: `furigana_map.txt`(77 条映射)

### 2. build_kfx.ps1 组装

运行 `build_kfx.ps1` 将参考模板 + 输入歌词 + 注音映射组装为 KFX 模板文件:

```
输出: *_KFX.ass (262 行)
├── [Script Info] ← 参考文件 lines 0-40
├── [V4+ Styles]  ← 参考文件样式 + Default 样式(手动补充)
├── [Events]
│   ├── code once × N   ← 参考文件 lines 43-122(全局变量、函数、形状)
│   ├── code line all    ← 行级计算
│   ├── code syl all     ← 音节级条件
│   ├── template × N     ← 各 Style 的效果模板
│   ├── template furi × 6 ← 注音显示模板(新增)
│   ├── Comment: karaoke × 63 ← 日语歌词源行(含 |< 注音)
│   └── Dialogue: OPCN × 63   ← 中文翻译行
```

#### 关键处理

| 处理项 | 说明 |
|--------|------|
| `\kf` → `\k` | LDDC 生成的 `\kf` 标签转为 Aegisub 识别的 `\k` |
| Style 分配 | 根据歌曲时间线 `Assign-Style` 函数将每行分配到正确段落 |
| 注音嵌入 | `Add-Furigana` 函数将 CJK 汉字替换为 `漢|<かん` 格式 |
| 上下文注音 | 同一汉字在不同词语中使用不同读音(一人→ひと, 出来→でき 等) |
| Comment 行 | 所有 karaoke 源行必须是 `Comment:` + `Effect=karaoke` |

### 3. Aegisub 执行模板

打开 `*_KFX.ass` → **自动化** → **卡拉OK模板** → 生成 ~31000+ 行 fx

### 4. 最终输出

约 31000+ 行 fx 效果行, 包含:
- 主文字效果(高亮、lead-in、lead-out)
- 粒子装饰(音符、羽毛、蝴蝶、花朵、草地等)
- 注音(furigana)显示
- 中文翻译行

## 遇到的问题与解决方案

### 问题 1: `unicode.len` 参数数量错误

**错误信息**:
```
Runtime error in template code: Expected 1 arguments, got 2
Code producing error: ci2 = { 0 }; cn = _G.unicode.len(orgline.text_stripped:gsub(" ",""))
```

**根因**: Lua 的 `string:gsub()` 返回 2 个值(结果字符串 + 替换次数), 而 `unicode.len()` 只接受 1 个参数。在某些旧版 Aegisub 中多余参数被忽略, 但 Aegisub 3.4.2 更严格。

**影响范围**: 参考文件中 5 处 `code line all` 行(OPJP 2 × 2, OPJP 4 × 1, OPJP 5 × 2), 以及点兔参考文件 1 处。

**修复方案**: 用括号包裹 `gsub()` 调用, 强制 Lua 只返回第一个值:

```lua
-- Before (错误):
cn = _G.unicode.len(orgline.text_stripped:gsub(" ",""))

-- After (正确):
cn = _G.unicode.len((orgline.text_stripped:gsub(" ","")))
```

**修复位置**: 同时修复了参考文件和 KFX 输出文件, 确保后续重建不会复现。

### 问题 2: Style not found: Default

**错误信息**: `WARNING: Style not found: Default`

**根因**: KFX 文件中有 `Dialogue: 0,...,Default,...` 分隔行(从参考文件复制而来), 但样式表中没有定义 `Default` 样式。

**修复方案**: 在样式表末尾添加一个最小化的 Default 样式定义:

```
Style: Default,Arial,20,&H00FFFFFF,&H000000FF,&H00000000,&H00000000,0,0,0,0,100,100,0,0,1,2,2,2,10,10,10,1
```

### 问题 3: 假名注音不显示

**现象**: KFX 源行中正确包含 `|<` 注音语法, 但执行模板后 fx 输出中没有 furigana 行。

**根因**: 原始参考模板中所有 template 行都使用了 `noblank` 修饰符, 这会跳过 furigana 音节。karaskel 解析 `|<` 后, furigana 数据存在于 `syl.isfuri=true` 的特殊音节中, 但没有模板去处理它们。

**参考**: `ヨスガノソラ/ツナグキズナ` 参考文件使用了 `template syl furi` 修饰符来同时处理主音节和 furigana 音节。

**第一次修复(不完美)**: 为 6 个 Style 各添加一条独立的 `template furi` 行:

```
Comment: 0,0:00:00.00,0:00:00.00,{Style},,0,0,0,template furi,!retime("line")!{\an5\pos($scenter,$smiddle)\fad(300,200)}
```

⚠️ **这种做法的问题**: 注音只是在整行时间内静态显示, 没有跟随主文字的卡拉OK特效(lead-in、highlight、lead-out)同步! 注音呆呆地显示在那里, 和主文字完全不同步。

**第二次修复(正确方案 ⭐模范案例)**: 使用 `template syl furi` 组合修饰符, 让注音和主文字共享同一套特效:

```ass
# 错误做法(注音不同步):
template noblank multi,{...高光效果...}     ← 只处理主文字
template furi,{...静态显示...}               ← 独立处理注音, 无特效同步

# 正确做法(注音同步 ✓):
template syl furi noblank multi,{...高光效果...}  ← 同时处理主文字和注音!
```

**关键知识点**:
- `syl furi` 是唯一允许的组合修饰符, 让 kara-templater 从**同一个模板**生成两个版本——一个给主文字(syl), 一个给注音(furi)
- 如果只写 `furi`(不写 `syl`), 模板变成 **furi-only 类**, 不再处理主文字!
- `$scenter`/`$smiddle` 内联变量会自动针对 furi 调整为注音的位置
- 装饰性效果(粒子、翅膀、花朵等 `notext` 模板)**不应加 furi**, 因为注音不需要装饰粒子

**修改的模板行数**: 16 行(6个 Style, 每个 2~3 个文字模板), 删除 6 行独立 `template furi`

### 问题 4: 缺少汉字注音映射「多」

**发现**: 「多分」(たぶん)一词中, 「分」的上下文覆盖正确生效(→ ぶん), 但「多」不在 `furigana_map.txt` 中。

**修复**: 在 `furigana_map.txt` 中补充 `多=た`。

### 问题 5: 注音分配不准确(多假名注到单字上)

**现象**: 部分多字词的注音被全部挂在一个汉字上, 而另一个汉字缺失注音。

**具体修正**:

| 词 | 修正前 | 修正后 | 说明 |
|---|---|---|---|
| 世界(せかい) | 世=せか, 界=い | 世=せ, 界=かい | 「せか」不是「世」的自然音节分割 |
| 必死(ひっし) | 必=ひ, 死=っし | 必=ひっ, 死=し | 促音「っ」属于前字 |
| 一人(ひとり) | 一=ひと, 人=无注音 | 一=ひと, 人=り | 人字缺失注音 |
| 出会(であう) | 出=で, 会=无注音 | 出=で, 会=あ | 会字缺失注音 |

**修复方式**: 修正 `furigana_map.txt` 中的默认映射, 新增 `人=り`、`会=あ` 映射。

### 问题 6: 高潮段中文遮挡日语注音

**现象**: OPJP 4(Chorus)段落的中文翻译行 Y=640, 与日语文字的 furigana 注音在垂直方向重叠。

**修复**: 将高潮段中文的 Y 坐标从 640 调整到 600, 给注音留出足够空间。

## 经验教训

1. **Aegisub 版本兼容性**: 旧模板在新版 Aegisub 上可能因 Lua 严格性不同而报错, 需检查 `gsub` 等多返回值函数的使用。
2. **⭐ Furigana 必须用 `syl furi` 组合**: 独立的 `template furi` 行只会让注音静态显示, 无法跟随主文字的卡拉OK特效同步。正确做法是在**所有文字模板**上使用 `template syl furi ...` 组合, 让注音和主文字共享同一套 lead-in/highlight/lead-out 效果。装饰模板(`notext`、粒子等)不加 `furi`。
3. **PowerShell 编码陷阱**: 日语字符串不能直接写在 `.ps1` 中, 必须用 Unicode 码点 `[char]0xXXXX` 或外部 UTF-8 文件。
4. **Default 样式**: 如果模板文件中有任何行引用了 `Default` 样式(即使只是分隔符), 样式表中必须有对应定义。
5. **`\kf` → `\k` 转换**: LDDC 输出的 `\kf` 标签需要转为 `\k`, Aegisub templater 只认 `\k`。
6. **注音分配要自然**: 多汉字词(如「世界」「必死」)的注音需要按自然音节分割到每个汉字上, 不要把多个假名全挤到一个字上而让另一个字没有注音。
7. **中文翻译位置**: 高潮段的中文翻译使用 `\pos` 手动定位时, Y 坐标要考虑日语行 + furigana 注音的高度, 避免遮挡。
