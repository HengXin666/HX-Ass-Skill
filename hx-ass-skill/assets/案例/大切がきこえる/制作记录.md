## 设计思路

### 与前两首的差异

这首歌**不再使用天使の3P模板**。前两首(羽ばたきのバースデイ、INNOCENT BLUE)属于元气可爱风, 使用了粉紫色 + PopJoy 字体 + 弹跳/闪烁效果。而「大切がきこえる」是一首**温暖柔和的抒情曲**, 需要截然不同的视觉风格:

| 对比项 | 前两首(天使の3P模板) | 大切がきこえる |
|--------|---------------------|---------------|
| 字体 | FOT-PopJoy Std B (可爱圆体) | FOT-NewCezanne Pro DB (优雅衬线) |
| 主色 | `&H00AF4BF0` 粉紫 | `&H00E0F0F5` 暖白 |
| 描边色 | 粉色系 | `&H00201A15` 深色 |
| 效果风格 | 弹跳、闪烁、粒子爆发 | 帘幕展开、碎片组装、羽毛飘散、齿轮微旋 |
| 情绪 | 欢快、元气 | 温暖、安静、感性 |

### 参考来源

从6个参考文件中筛选, 最终选择两个来源的混合:

1. **Made in Abyss OP** — `\clip` 帘幕展开效果: 优雅、安静、适合开场/收束
2. **命运石之门 ファティマ** — 7种效果中选取4种:
   - JP 2: 碎片组装(4象限 `\clip` + 3D旋转飞入)
   - JP 3: 静态 + 微型齿轮旋转(`\fad(300,300)`)
   - JP: 下落 + 羽毛飘散(5个羽毛粒子/音节)
   - JP 5: 简单淡入淡出 + 棍棒旋转 + 雪花数字

被排除的参考: 点兔ポポロン(太可爱)、Charlotte Bravely You(手工制作不可复用)、ヨスガノソラ(扫描高亮风格不匹配)

### 2. 构建脚本 (`build_kfx_taisetsu.ps1`)

**关键特点**: 完全自包含, 不引用外部参考模板文件。所有 Script Info、Styles、template 块均在脚本内联生成。

```
输出: *_KFX.ass (173行)
├── [Script Info]        ← 脚本内联定义
├── [V4+ Styles]         ← 15个样式(7个JP + CN + Default, 各含 -furigana 变体)
├── [Events]
│   ├── code syl all × 2     ← firstsyl/lastsyl 标记
│   ├── OPJP template × 4    ← Intro: 帘幕展开(左右半)
│   ├── OPJP 2 template × 5  ← Verse: 碎片组装(4象限 + syl2end)
│   ├── OPJP 3 code+template × 3 ← Pre-chorus: 齿轮 + 静态
│   ├── OPJP 4 code+template × 4 ← Chorus: 下落 + 羽毛
│   ├── OPJP 5 code+template × 4 ← Interlude: 淡入淡出 + 棍棒 + 雪花
│   ├── OPJP 6 template × 4  ← Outro: 慢速帘幕(800ms)
│   ├── OPCN template × 4    ← 中文: 帘幕展开
│   ├── Comment: karaoke × 50 ← 日语歌词(含 |< 注音)
│   └── Dialogue: OPCN × 50  ← 中文翻译
```

### Assign-Style 时间阈值

```powershell
if ($ms -lt 26000)  { return "OPJP" }     # Intro → 26380ms
if ($ms -lt 51000)  { return "OPJP 2" }   # Verse 1 → 51390ms
if ($ms -lt 57800)  { return "OPJP 3" }   # Pre-chorus 1 → 57900ms
if ($ms -lt 86000)  { return "OPJP 4" }   # Chorus 1 → 89860ms
if ($ms -lt 96000)  { return "OPJP 5" }   # Interlude → 96460ms
if ($ms -lt 110000) { return "OPJP 2" }   # Verse 2 → 110110ms
if ($ms -lt 126000) { return "OPJP 3" }   # Pre-chorus 2 → 126460ms
if ($ms -lt 142000) { return "OPJP 2" }   # Transition → 142370ms
if ($ms -lt 152000) { return "OPJP 2" }   # Bridge active → 152220ms
if ($ms -lt 190000) { return "OPJP 3" }   # Bridge narration → 190480ms
if ($ms -lt 208000) { return "OPJP 4" }   # Final Chorus → 208410ms
if ($ms -lt 224000) { return "OPJP 5" }   # Coda → 224760ms
return "OPJP 6"                            # Outro
```

### 设计决策

1. **Bridge 分两段**: 前半(しゃべり足りない～歌おう)节奏活跃用碎片组装, 后半(付きあうよ～いっぱい)叙事温柔用齿轮微旋
2. **Coda 使用 OPJP 5**: 耳すませば～きこえる音 与 Interlude 的キラキラ呼应, 都是"倾听"主题
3. **Outro 使用 OPJP 6(慢速帘幕)**: 800ms 动画(比 Intro 的 600ms 更慢), 形成首尾呼应, 同时暗示余韵
4. **OPJP 不设 OPJP 6**: Intro 只有两行歌词, 使用 OPJP(600ms帘幕)足够

## 遇到的问题与解决

### 1. PowerShell UTF-8 编码问题

**问题**: PS1 文件保存为 UTF-8(无 BOM)时, PowerShell 按系统默认编码(GBK)读取, 导致 CJK 字符损坏, `&H00E0F0F5` 中的 `&` 被解析为 PS 运算符, 报错 exit code 4294770688。

**解决**: 创建 `_fix_encoding.py` 将文件转换为 UTF-8 BOM 编码; 创建 `_run_build.py` 通过 Python subprocess 运行脚本并正确捕获输出。

### 2. BOM + `#` 注释冲突

**问题**: UTF-8 BOM 前缀 `﻿` 与第一行 `##` 注释结合, 产生 `﻿##` 命令无法识别警告。

**解决**: 将 `##` 改为 `#`。警告仍存在但不影响脚本执行(RC=0)。

### 3. Assign-Style 时间阈值计算错误(3轮修正)

**问题**: 将 `分:秒` 格式直接拼接为毫秒数时, 多次混淆 `m:ss` 和实际毫秒值:
- 第1轮: `126000` 误认为 1:26(实为 2:06), Chorus 1 结束阈值过大
- 第2轮: `136000` 误认为 1:36(实为 2:16), Interlude 结束阈值过大
- 第3轮: `310000` 误认为 3:10(实为 5:10), Bridge narration 阈值过大

**解决**: 逐行提取源文件所有 `orig` 行的精确毫秒时间戳, 完全重建阈值表。关键公式: `m:ss.cc` → `m*60000 + ss*1000 + cc*10`。

**教训**: **永远从源文件提取实际时间戳, 不要心算 `分:秒` → 毫秒转换。**

## 经验总结

1. **不同歌曲风格需要不同模板**: 同一动画的歌曲也可能风格迥异, 不能盲目复用。需分析歌曲情绪后选择合适的参考。
2. **混合参考源**: 从多个参考模板中各取所长(Made in Abyss 的优雅 + 命运石之门的变化), 效果比单一来源更好。
3. **自包含脚本**: 不依赖外部参考文件, 所有模板内联在脚本中, 更便于维护和调试。
4. **时间戳精确性**: Assign-Style 的阈值必须基于源文件的实际时间戳计算, 手动心算容易出错(经历了3轮修正)。
5. **歌曲结构的灰色地带**: Bridge 段落可能需要分成"活跃"和"叙事"两部分, 使用不同效果。
