# OPED 歌词特效设计思路

> 来源: Seekladoom 的设计哲学 + 实战经验总结
> 用途: AI 在制作 KFX 时的风格决策依据

---

## 一、设计流派分类

### 1. 可直接从 OPED 画面中观察到的设计元素

| 流派 | 说明 | 适合场景 |
|------|------|---------|
| **STAFF流** | 从 OP/ED 的 STAFF 字幕设计中提取灵感(字体、颜色、排版、动效) | 任何有明确 STAFF 设计的 OP/ED |
| **LOGO流** | 从作品 LOGO 的设计语言中提取(字体选择、颜色方案、特效风格) | LOGO 有明显设计特征的作品 |
| **道具流** | 从画面中出现的标志性道具提取元素(武器、乐器、魔法阵等) | 有明显标志性道具的作品 |
| **人物流** | 以人物形象/应援色/表情动作作为字体和特效的设计依据 | 角色特征鲜明的作品 |

### 2. OPED 中不一定能直接看到, 但可以挖掘的

| 流派 | 说明 | 示例 |
|------|------|------|
| **歌词含义流** | 根据歌词内容/情感选择特效(悲伤→雨滴、希望→光芒) | 大切がきこえる → 温暖clip+羽毛 |
| **题材流** | 根据作品题材选择字体风格化方向(科幻→方正、魔法→花体) | 命运石之门 → 齿轮+碎片+科技感 |
| **创意风格流** | 完全脱离 OPED 画面, 自由创作(包括无视 OP 做的风格) | 天气之子 GRAND ESCAPE → 雨水粒子系统 |
| **技术流** | 以展示特定技术能力为核心(复杂绘图、多层叠加等) | 假面骑士 → glitter闪光函数 |
| **正片元素流** | OPED 里没有但正片或原作里有的元素 | 原作道具、标志性场景元素 |

### 3. 打破常规的思维

| 手法 | 说明 | 何时使用 |
|------|------|---------|
| **不匹配做法** | 故意选择与作品氛围不匹配的字体/特效, 形成反差 | 需要制造戏剧性冲突时 |
| **不讨喜的选择** | 故意选让人不喜欢的字体和特效, 服务于特定表达 | 表达不安/压抑/诡异氛围时 |
| **不按套路出牌** | 其他任何违反常规的做法 | 需要出其不意的创意时 |

### 4. 风格转化与融合

| 手法 | 说明 | 实例 |
|------|------|------|
| **LOGO→STAFF** | 把 LOGO 的颜色和转场效果用在 STAFF 字体上 | 统一视觉风格 |
| **应援色→STAFF** | 人物/武器的颜色用在 STAFF 字体上 | 每位角色有专属色彩 |
| **字体=人物** | 以字体来象征人物或其他事物 | COP CRAFT OP 女主变装段 |
| **跨作品融合** | 从不同作品提取元素混搭 | 大切がきこえる = MIA clip + S;G 碎片 |

## 二、风格-情感匹配表(AI 决策用)

这是 AI 选择特效时的核心参考表。根据歌曲情感选择对应的特效组合:

### 情感光谱

```
冷静/空灵 ←————————————→ 热血/燃烧
   │                           │
   ├─ clip帘幕展开              ├─ glitter闪光粒子
   ├─ 简单淡入淡出              ├─ 三角碎片爆发
   ├─ 齿轮微旋                  ├─ 碎片组装飞入
   ├─ 棍棒旋转                  ├─ 雨滴粒子暴风
   ├─ 雪花数字                  ├─ 换色高亮卡拉OK
   └─ 羽毛轻柔飘散              └─ 多层叠加重组
```

### 推荐组合

| 情感 | 主效果 | 辅助效果 | 入场 | 退场 | 参考作品 |
|------|--------|---------|------|------|---------|
| **温暖治愈** | clip帘幕 | 羽毛飘散 | 从中心展开 | 淡出 | 大切がきこえる |
| **科幻悬疑** | 碎片组装 | 齿轮旋转 | 四象限飞入 | 碎裂消散 | 命运石之门 0 |
| **热血激燃** | glitter闪光 | 三角碎片 | 逐字入场+颤动 | 粒子爆发 | 假面骑士 EX-AID |
| **抒情感伤** | 雨滴下落 | 光晕圆点 | 从上方落入 | 淡出溶解 | 天气之子 GRAND ESCAPE |
| **欢快可爱** | 逐字弹跳 | 星星/爱心 | 上下交替入场 | 缩小消失 | 天使の3P |
| **庄严史诗** | 横向clip扫描 | 数字雪花 | 从左到右展开 | 缓慢淡出 | Little Busters! |
| **日常平静** | 简单淡入淡出 | 无 | `\fad(200,200)` | `\fad(200,200)` | 量产模板 |
| **神秘诡异** | iclip字体分割 | 随机抖动 | 突然出现 | 突然消失 | 未指定 |

## 三、段落-特效映射策略

一首完整歌曲的不同段落应使用不同特效, 形成 **视觉叙事弧线**:

```
Intro (前奏/引入)
  └── 效果: 最简单、最安静 → clip帘幕 / 简单淡入
  └── 作用: 建立基调, 不喧宾夺主

Verse (主歌)
  └── 效果: 中等复杂度 → 碎片组装 / 逐字入场
  └── 作用: 推进叙事, 展现歌词内容

Pre-chorus (副歌前)
  └── 效果: 开始升温 → 齿轮旋转 / 微动效果
  └── 作用: 蓄势待发, 为高潮做准备

Chorus (副歌)
  └── 效果: 最复杂、最华丽 → 粒子爆发 / 多层叠加
  └── 作用: 情感高潮, 全力释放

Bridge (桥段)
  └── 效果: 变化 → 不同于前面所有段落
  └── 作用: 打破重复, 制造新鲜感

Coda (尾奏)
  └── 效果: 回归平静 → 简单淡入淡出 / 棍棒旋转
  └── 作用: 收束情感, 余韵悠长

Outro (结尾)
  └── 效果: 最安静 → 慢速clip / 单纯淡出
  └── 作用: 画上句号
```

## 四、技术要点速查

### 4.1 template 行关键字

| 关键字 | 作用 |
|--------|------|
| `template` | 声明模板行 |
| `char` | 逐字符处理 |
| `syl` | 逐音节处理 |
| `noblank` | 跳过空白音节 |
| `notext` | 只生成辅助元素(粒子等), 不生成文字 |
| `multi` | 多行匹配 |
| `loop N` | 循环 N 次(用于粒子倍增) |
| `fxgroup name` | 条件分组 |

### 4.2 retime 函数

| 调用 | 含义 |
|------|------|
| `retime("syl",A,B)` | 相对于音节起点, 偏移 [A, B] |
| `retime("syl2end",A,B)` | 从音节起点到行尾, 偏移 [A, B] |
| `retime("start2syl",A,B)` | 从行首到音节起点, 偏移 [A, B] |
| `retime("line",A,B)` | 相对于行起止, 偏移 [A, B] |
| `retime("postline",A,B)` | 行尾之后 [A, B] |

### 4.3 常用坐标变量

| 变量 | 含义 |
|------|------|
| `$center` / `$middle` | 当前音节中心坐标 |
| `$scenter` / `$smiddle` | 当前音节中心(syl级) |
| `$sleft` / `$sright` | 音节左右边界 |
| `$lleft` / `$lright` | 行的左右边界 |
| `$width` / `$height` | 音节宽高 |
| `$lwidth` / `$lheight` | 行宽高 |
| `$top` / `$bottom` | 音节顶底 |
| `$si` | 音节索引(1-based) |
| `$syln` | 音节总数 |
| `$sdur` / `$ldur` / `$dur` | 音节/行/当前时长(ms) |
| `j` / `maxj` | loop 当前/最大值 |

## 五、制作工作流

```
1. 分析歌曲 → 确定情感基调 → 查"风格-情感匹配表"
2. 划分段落 → Intro/Verse/Chorus/Bridge/Outro
3. 为每个段落分配特效 → 查"段落-特效映射策略"
4. 从素材库选取模板 → doc/素材库/对应分类/
5. 调整颜色/字体/速度参数 → 匹配作品风格
6. 构建 Assign-Style 函数 → 精确毫秒阈值
7. 生成 KFX → 在 Aegisub 中运行 karaoke templater
```
