# 注音系统 (Furigana) 完整指南

> 为日语歌词中的汉字添加假名注音是高质量 KFX 的必选项。

---

## 一、Aegisub 原生注音语法

### `|<` 语法

在 karaoke 源行中内嵌注音：

```
{\k20}合|<あ{\k19}図|<い{\k23}は
```

含义：「合」读「あ」，「図」读「い」。Templater 处理时自动生成 `-furigana` 后缀样式。

### 注音映射文件 (`furigana_map.txt`)

项目根目录，UTF-8 编码，全歌共享：

```
始=はじ
合=あ
図=い
世=せ
界=かい
飛=と
```

格式：`漢字=ひらがな読み`，每行一个映射。当前 218+ 条。

## 二、上下文相关注音（重点！）

同一个汉字在不同词语中可能有不同读音。必须通过检查前后文进行覆盖：

### 已积累的覆盖规则

| 汉字 | 默认 | 上下文 | 覆盖后 | 判断条件 |
|------|------|--------|--------|---------|
| 一 | いっ | 一人 | ひと | 后接「人」 |
| 出 | だ | 出来/出会/出口 | で | 后接「来」「会」「口」 |
| 来 | らい | 出来る | き | 前接「出」 |
| 会 | あ | 出会う | — | 前接「出」 |
| 分 | わ | 自分/多分 | ぶん | 前接「自」「多」 |
| 笑 | わら | 笑顔 | え | 后接「顔」 |
| 顔 | かお | 笑顔 | がお | 前接「笑」 |
| 何 | なに | 何回 | なん | 后接「回」 |
| 回 | まわ | 何回 | かい | 前接「何」 |
| 今 | いま | 今日 | きょ | 后接「日」 |
| 日 | ひ | 今日 | う | 前接「今」 |
| 日 | ひ | 日常 | にち | 后接「常」 |
| 日 | ひ | 明日 | た | 前接「明」 |
| 音 | おと | 音色 | ね | 后接「色」 |
| 音 | おと | 鐘の音 | ね | 后接特定字 |
| 風 | かぜ | 風に(ふうに) | ふう | 后接「に」 |
| 大 | だい | 大切 | たい | 后接「切」 |
| 声 | こえ | 歌声 | ごえ | 前接「歌」 |

### PowerShell 中的实现

**不要直接写日语字符**，使用 Unicode 码点：

```powershell
if ($code -eq 0x4E00 -and $plain.Contains([string][char]0x4E00+[string][char]0x4EBA)) {
    $r = [string][char]0x3072+[string][char]0x3068  # 一人 → ひと
}
```

## 三、注音分配原则

### 自然音节分割

多汉字词的注音必须按自然音节分割到每个汉字上：

| 词 | ✅ 正确 | ❌ 错误 |
|---|---|---|
| 世界(せかい) | 世=せ, 界=かい | 世=せか, 界=い |
| 必死(ひっし) | 必=ひっ, 死=し | 必=ひ, 死=っし |
| 一人(ひとり) | 一=ひと, 人=り | 一=ひと, 人=无注音 |
| 出会(であう) | 出=で, 会=あ | 出=であ, 会=う |
| 圧倒的(あっとうてき) | 圧=あっ, 倒=とう, 的=てき | 全挂一字 |

### 规则

1. **促音「っ」属于前字**（如 必死 → 必=ひっ）
2. **每个汉字都必须有对应注音**，不要遗漏
3. 1个汉字可对应多个假名（如 蹲=うずくま，4假名）
4. **⚠️ 注音必须是假名（ひらがな），绝对不能是罗马音**
5. 纯假名（ひらがな/カタカナ）不需要注音
6. 只对 CJK 汉字（U+4E00~U+9FFF）添加注音

## 四、⭐ 注音特效同步 (`syl furi`)

### 核心原则

注音必须和主文字的卡拉OK特效（lead-in、highlight、lead-out）**完全同步**。

### ❌ 错误做法：独立 `template furi`

```ass
template noblank multi,{...高光效果...}     ← 只处理主文字
template furi,{...静态显示...}               ← 注音只是呆呆显示，无同步
```

### ✅ 正确做法：`template syl furi` 组合

```ass
template syl furi noblank multi,{...高光效果...}  ← 同时处理主文字和注音!
```

`syl furi` 让 kara-templater 从**同一个模板**生成两个版本：
- syl 版本：给主文字，`$scenter`/`$smiddle` 为主文字位置
- furi 版本：给注音，`$scenter`/`$smiddle` 自动调整为注音位置

### 修改规则

| 模板类型 | 是否加 furi | 原因 |
|---------|:---:|------|
| 文字高光 `template noblank multi` | ✅ | 注音需同步高亮 |
| 文字 lead-in `template char` | ✅ | 注音需同步入场 |
| 文字 lead-out `template noblank char` | ✅ | 注音需同步退场 |
| 粒子/装饰 `template notext` | ❌ | 装饰不给注音 |
| 矢量绘图 `\p1`/`\p2`/`\p3` | ❌ | 绘图不适用注音 |

### ⚠️ 注意事项

1. 只写 `furi` 不写 `syl` → 模板变成 furi-only，**不再处理主文字**！
2. `syl furi` 是唯一组合 — 不能写 `char furi`（应写 `syl furi char`）
3. `code syl all` 行不需要改为 `code syl furi all`

## 五、遇到新歌时的注音工作流

1. 加载已有 `furigana_map.txt`
2. 扫描新歌歌词中的所有 CJK 汉字
3. 标记未映射的新汉字
4. 查询正确读音并添加到映射文件
5. 检查是否需要新的上下文覆盖规则
6. 生成带 `|<` 注音的 karaoke 源行
7. 构建后验证：在 Aegisub 执行后检查 furigana 行是否正确显示
